# 📝 LM Studio における Intel Arc A770 系 GPU の対応状況・トラブルシューティングまとめ  
（GitHub Issue #379 の内容に基づく）

## 🎯 概要  
LM Studio 0.3.9（Build 5）以降、Intel Arc A770 16GB を中心とした Arc GPU が **Vulkan llama.cpp runtime で認識されない** という報告が複数寄せられた。  
以前のバージョンでは正常に動作していたユーザーも多く、アップデート後に突然 GPU が 0 枚として扱われるケースが発生した。

---

# 🔍 主な症状  
- **GPU が 0 枚として認識される**  
- Mission Control → Hardware にて **0 GPUs detected / 0 VRAM**  
- CPU モードでは動作するが、GPU モードではモデルがロードできない  
- llama.cpp の Vulkan runtime 更新後に発生した可能性が指摘される  

---

# 🛠️ 行われた主な対応と効果

## 1. **LM Studio のアップデート（0.3.9 b6）**
- 開発者から「0.3.9 b6 で修正されている可能性がある」と案内  
- しかし、**報告者は改善せず**。

---

## 2. **Intel Arc GPU ドライバーのロールバック**
- ドライバー **101.6557 → 101.6460** に戻したユーザーが  
  **GPU 認識が復活した** と報告  
- これにより、**原因の一部はドライバー側の問題**である可能性が高まった。

---

## 3. **LM Studio を「管理者権限で実行」**
複数ユーザーが以下の報告をしている：

- **管理者権限で起動したら GPU が認識された**  
- 以前は不要だったが、0.3.9（Build 5）以降で必要になったケースがある  
- A770M や A580 のユーザーも同様の改善を確認  

これは Issue 内で最も多く成功報告が集まった対処法。

---

## 4. **llama.cpp runtime の更新**
- 0.3.9-6 にアップデート後、  
  **管理者権限で起動 → llama.cpp を最新に更新 → GPU が認識された**  
  という成功例あり。

---

## 5. **LM Studio 0.3.10（Build 4）での完全修正**
Issue の投稿者（Slurpeeguy）が最終的に：

> **Version 0.3.10 (Build 4) で全ての問題が解決した**

と報告し、Issue はクローズされた。

---

# 🎉 実際に解決したユーザーの主なパターン

| 解決方法 | 成功報告数 | 備考 |
|---------|-----------|------|
| **管理者権限で LM Studio を起動** | 多数 | 最も効果が高い |
| **GPU ドライバーを 101.6460 にロールバック** | 複数 | 101.6557 で問題発生 |
| **LM Studio 0.3.10 (Build 4) に更新** | 公式に解決 | Issue 投稿者が確認 |
| **llama.cpp runtime を最新に更新** | 一部 | 管理者権限と併用で成功 |

---

# 📌 補足：未解決のケース  
- GPU は認識されるが、**モデルをロードすると失敗する**という別の問題も報告されている  
- Stable Diffusion は動作するが LM Studio のみ失敗する例もあり  
- これらは Issue 内では未解決のまま

# 🧭 まとめ  
Intel Arc A770 系 GPU が LM Studio で認識されない問題は、  
**ドライバーの相性**と **LM Studio の権限周りの変更**が複合的に影響していたと考えられる。

最終的には：

- **LM Studio 0.3.10（Build 4）で公式に修正**  
- それ以前の暫定対処としては  
  - 管理者権限で起動  
  - ドライバーのロールバック  
  が有効だった。

Arc GPU ユーザーにとっては重要な改善であり、今後も Vulkan runtime の更新に伴う挙動変化には注意が必要。

# 📝 LM Studio × Intel Arc A770  
# Web 上で確認できた追加の問題・トラブルシューティング レポート  
（GitHub Issue #379 以外の情報を含む）

---

# 1. Linux（Ubuntu）環境では A770 が比較的安定して動作  
**Ubuntu 24.04 + LM Studio 0.3.13-2** の環境では、Arc A770 が特別な設定なしで Vulkan バックエンドとして自動認識され、問題なく動作したという報告がある。

### 🔍 ポイント
- AppImage を実行するだけで GPU が認識  
- 特別なドライバー設定は不要  
- Windows での不具合と対照的に、Linux では安定している例が多い

### 💡 考察  
Windows 版 LM Studio の Vulkan llama.cpp runtime との相性問題が、A770 の不具合の主要因である可能性が高い。

---

# 2. A770 では「7B モデルが限界」という性能面の制約  
Intel Arc A770 16GB を使ってローカル LLM を動かしたユーザーから、**VRAM 16GB でも 7B モデルが限界**という報告がある。

### 🔍 内容
- 104B モデルは当然不可  
- 13B 以上のモデルも実質的に厳しい  
- VRAM 16GB でも llama.cpp の最適化が不十分なため性能が出ない

### 💡 技術的背景  
Arc GPU の XMX コアを llama.cpp が十分に活用できていない可能性がある。

---

# 3. A770 の LLM 推論性能が期待より低い  
A770 の LLM 推論性能を検証した動画レビューでは、**Quinn 3 14B が 13 tokens/s と非常に低速**だったという報告がある。

### 🔍 問題点
- 期待値（30〜40 tok/s）を大きく下回る  
- Vega 64 より遅いという指摘  
- XMX コアを使うために **Intel OneAPI Base Toolkit** と **Visual Studio 2019** のインストールが必要で、セットアップが複雑

### 💡 考察  
Arc GPU の LLM 推論は、現状では最適化が不十分で、  
**「動くが速くはない」**という評価が多い。

---

# 4. Intel Arc ドライバー更新による環境破損と再構築例  
Arc A770 のドライバー設定を変更した結果、GPU 環境が壊れ、再構築が必要になった例がある。

### 🔍 発生した問題
- 「実行する権限がありません」エラー  
- ドライバー更新後に GPU が正常動作しなくなる  
- 再インストールで復旧

### 💡 LM Studio との関連性  
LM Studio でもドライバーのバージョンによって GPU 認識が変わる例があるため、  
**Arc ドライバーの更新は慎重に行う必要がある**。

---

# 5. llama.cpp を直接使用した場合の Arc A770 の注意点  
Arc A770 で llama.cpp を直接動かしたユーザーから、以下のような注意点が報告されている。

### 🔍 注意点
- OneAPI Base Toolkit のインストールが必須  
- CLBlast を使う OpenCL バックエンドで動作  
- llama.cpp のバージョンによって動作が不安定  
- Windows では特に環境構築が複雑

### 💡 LM Studio への示唆  
LM Studio も内部で llama.cpp を利用しているため、  
**llama.cpp のバージョン更新が Arc GPU の動作に影響する**可能性が高い。

---

# 6. 画像生成モデルは LM Studio では動作しない（仕様）  
LM Studio は **テキスト LLM 専用**であり、画像生成モデル（Diffusion 系）はロードできないという報告がある。

### 🔍 影響
- A770 の GPU を使って画像生成 LLM を動かそうとして失敗する例がある  
- これは LM Studio の仕様であり、A770 の問題ではない

---

## 🟨 性能面の制約
